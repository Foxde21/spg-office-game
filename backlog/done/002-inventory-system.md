# Система инвентаря

## Описание
Реализовать инвентарь игрока: сбор предметов, использование, отображение в UI.

## Definition of Ready (DOR)
- [x] Задача четко сформулирована
- [x] Определены критерии приемки
- [x] Оценена сложность (Story Points)
- [x] Нет блокирующих зависимостей
- [x] Определены необходимые ресурсы/ассеты

## Definition of Done (DOD)
- [x] Код написан и работает
- [x] Код соответствует стайлгайду
- [x] Функционал протестирован вручную
- [x] Нет консольных ошибок
- [x] Документация обновлена (если требуется)
- [ ] Код прошел ревью (если требуется)

## Критерии приемки
1. [x] Можно подобрать предмет (клавиша E)
2. [x] Предмет появляется в инвентаре
3. [x] UI инвентаря открывается/закрывается (клавиша I)
4. [x] Можно использовать предмет
5. [ ] Предметы можно давать NPC (в следующей задаче с квестами)

## Технические детали
- Создать `InventoryManager` класс
- UI: сетка слотов (максимум 16)
- Типы предметов: квестовые, расходники, документы
- Предметы как интерактивные объекты на карте

## Зависимости
- Нет

## Оценка
- Story Points: 5
- Приоритет: High
