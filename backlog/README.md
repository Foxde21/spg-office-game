# BACKLOG - Office Quest Game

## Структура

- **todo/** — задачи, готовые к взятию в работу (прошли DOR)
- **in-progress/** — задачи в работе
- **done/** — выполненные задачи (прошли DOD)

## Workflow

1. Создаем тикет по шаблону `TEMPLATE.md`
2. Заполняем DOR — проверяем готовность
3. Перемещаем в `todo/`
4. **Создаем ветку от dev:** `git checkout dev && git pull && git checkout -b feature/<ticket-id>-<description>`
5. Берем в работу → перемещаем в `in-progress/`
6. Выполняем все пункты DOD
7. Коммитим в ветку (НЕ в main/dev!)
8. Push + PR в dev
9. После ревью мержим в dev
10. Перемещаем в `done/`
11. Релиз: PR из dev в main

## Git Branches

**Структура веток:**
- `main` — релизы (защищен)
- `dev` — разработка (защищен)
- `feature/*` — фичи

**Каждая задача — в отдельной ветке от dev!**

```
feature/001-quest-system
feature/002-inventory-system
bugfix/003-dialogue-fix
refactor/004-player-movement
```

**Запрещено коммитить напрямую в main и dev!**

## Приоритеты

- **P0 (Critical)** — блокирует релиз/геймплей
- **P1 (High)** — важный функционал
- **P2 (Medium)** — улучшения
- **P3 (Low)** — nice to have

## Story Points

- **1 SP** — простая задача, < 1 час
- **2 SP** — небольшая задача, 1-2 часа
- **3 SP** — средняя задача, 2-4 часа
- **5 SP** — сложная задача, полдня
- **8 SP** — большая задача, день
- **13 SP** — эпик, нужно декомпозировать

## Метки

### Тип
- `feature` — новый функционал
- `bug` — исправление ошибок
- `refactor` — рефакторинг
- `docs` — документация
- `assets` — ассеты (графика, звук)

### Компонент
- `core` — ядро игры
- `gameplay` — игровые механики
- `ui` — интерфейс
- `audio` — звук и музыка
- `assets` — ассеты
