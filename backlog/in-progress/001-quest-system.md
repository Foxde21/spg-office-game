# Система квестов

## Описание
Реализовать полноценную систему квестов: получение заданий, отслеживание прогресса, выполнение, награды (уважение, стресс).

## Definition of Ready (DOR)
- [x] Задача четко сформулирована
- [x] Определены критерии приемки
- [x] Оценена сложность (Story Points)
- [x] Нет блокирующих зависимостей
- [x] Определены необходимые ресурсы/ассеты

## Definition of Done (DOD)
- [x] Код написан и работает
- [x] Код соответствует стайлгайду
- [x] Функционал протестирован вручную
- [x] Нет консольных ошибок
- [x] Документация обновлена (если требуется)
- [ ] Код прошел ревью (если требуется)

## Критерии приемки
1. [x] NPC может выдавать квест
2. [x] Квест отображается в UI (список активных)
3. [x] Можно отследить прогресс квеста
4. [x] При выполнении — награда (уважение +/-)
5. [x] Выполненные квесты сохраняются

## Технические детали
- Создать `QuestManager` класс
- Хранить квесты в GameState
- UI: панель активных квестов (справа)
- Триггеры: диалоги, предметы, локации

## Зависимости
- Нет

## Оценка
- Story Points: 5
- Приоритет: High
